## 3. 개발 원칙 (Development Principles)

### 🔒 보안 원칙

1. 해킹 위험이 있는 코드를 생성하지 않는다.
2. 개인정보 유출 우려가 있는 패턴을 사용하지 않는다.
3. 입력값 검증, 버퍼 오버플로우 방지, 메모리 안전성을 항상 확보한다.
4. 암호화/인증/권한 관리는 업계 표준을 따른다.
5. 보안 취약점이 의심되면 반드시 팀 리드에게 보고한다.

### ⚡ 성능 원칙

1. 단순히 동작하는 코드가 아닌, 하드웨어 자원을 최소화하는 코드를 작성한다.
2. 시간 복잡도와 공간 복잡도를 항상 고려한다.
3. 알고리즘 선택 시 전체 아키텍처 차원에서 최적의 선택을 한다.
4. 불필요한 메모리 할당, 불필요한 복사, 불필요한 I/O를 제거한다.

### 🏗️ 아키텍처 원칙

1. 확장 가능하고 유지보수가 용이한 구조를 설계한다.
2. 모듈 간 결합도를 낮추고 응집도를 높인다.
3. 적절한 디자인 패턴을 적용하되, 과도한 추상화를 피한다.
4. 기술 부채를 최소화하는 방향으로 의사결정한다.

### 📋 코드 품질 원칙

1. 자기 문서화(self-documenting) 코드를 지향한다.
2. 의미 있는 변수명, 함수명을 사용한다.
3. 단위 테스트 작성을 기본으로 한다.
4. 코드 리뷰에서 지적받을 만한 코드를 사전에 방지한다.
