# mcjo.pro 확인

팀 리드(mcjo)의 비동기 지시사항 파일 `C:\George-Kimi\mcjo.pro`를 읽고 처리합니다.

## 실행 절차

### 1. 처리 시작 시간 기록
```bash
date +"%Y-%m-%d %H:%M:%S"
```
이 시간을 `처리시작시간`으로 기록한다.

### 2. mcjo.pro 파일 읽기 (계층 구조)

**우선순위 로직** (자동 순회):
1. `C:\George-Kimi\project*` 폴터를 모두 찾아서 번호 순으로 정렬 (내림차순)
   - 예: project004 → project003 → project002 → project001
2. 각 `project{N}\mcjo.pro` 파일을 순서대로 확인:
   - [INPUT] 섹션에 지시사항이 있는지 확인
   - 지시사항이 있으면 **해당 파일 사용** (처리 후 순회 중단)
   - 없으면 다음 project로 이동
3. 모든 project의 mcjo.pro에 지시사항이 없으면:
   - 메인 `C:\George-Kimi\mcjo.pro` 확인

**동작 방식**:
- `/mcjo` 실행 시 **항상 모든 project 폴터를 자동으로 순회**
- 가장 최신 project (번호가 가장 큰 것)부터 우선 확인
- 현재 작업 디렉토리와 무관하게 동작

**파일 구조**:
- `C:\George-Kimi\mcjo.pro` — 메인 (죠지 학습, 간단한 질문)
- `C:\George-Kimi\project004\mcjo.pro` — project004 전용
- `C:\George-Kimi\project{N}\mcjo.pro` — 각 프로젝트 전용

**주의**: 프로젝트 mcjo.pro가 최우선이며, HISTORY는 각각 독립적으로 관리됩니다.

### 3. [INPUT] 섹션에서 새 지시사항 확인
`>>>` 마커 아래의 내용을 파싱한다. 내용이 없으면 "새 지시사항 없음"을 보고한다.

### 4. 지시사항 처리
입력된 지시사항을 프롬프트 직접 입력과 동일하게 처리한다.
- 질문이면 답변
- 작업 지시면 계획 수립 후 실행
- 복수 항목이면 순서대로 처리

### 5. 처리 완료 시간 기록
```bash
date +"%Y-%m-%d %H:%M:%S"
```
이 시간을 `처리완료시간`으로 기록한다.

### 6. mcjo.pro 파일 업데이트
처리 완료 후 파일을 업데이트한다:
- 처리된 입력을 `[HISTORY]` 섹션으로 이동
- `[INPUT]` 섹션의 `>>>` 마커를 비워서 다음 입력 대기 상태로 복원

### 7. 처리 결과 보고
처리 결과를 프롬프트에 출력한다.

## [HISTORY] 마킹 형식
```
# [DONE] 입력내용 한줄 요약
#   입력: YYYY-MM-DD HH:MM:SS | 완료: YYYY-MM-DD HH:MM:SS | 소요: Xm Xs
#   모델: claude-sonnet-4-5 | 토큰: in: {입력토큰}, out: {출력토큰}
#   프롬프트:
#     {팀 리드가 작성한 원본 프롬프트 전문 — 줄바꿈 유지, 각 줄 앞에 #     접두사}
#   → 처리 결과 요약
```

### 형식 규칙

- **입력시간**: /mc 스킬이 호출된 시점 (처리시작시간)
- **완료시간**: mcjo.pro 파일 업데이트 직전 시점 (처리완료시간)
- **소요시간**: 완료 - 입력 시간 차이
- **모델**: 현재 사용 중인 모델명 (claude-sonnet-4-5)
- **토큰**: 해당 /mc 처리에 사용된 실제 토큰량 (in: 입력, out: 출력)
  - 서브에이전트 사용 시: TaskOutput의 total_tokens 합산 포함
  - 메인 대화 토큰: statusline 참조하여 처리 전후 차이 계산
  - 정확한 수치 확인 불가 시: "~추정값" 으로 표기
- **프롬프트**: 팀 리드가 작성한 원본 입력 전문을 그대로 보존 (요약/축약 금지)
