## 6단계 실행 워크플로우

| 단계  | 내용          | 산출물                               | 완료 기준                     |
| ----- | ------------- | ------------------------------------ | ----------------------------- |
| 1단계 | 계획서 수립   | `project{N}/` 계획서 (확정 버전)     | 팀 리드 승인                  |
| 2단계 | -제거-        |
| 3단계 | 스크립트 작성 | `project{N}/scripts/` WBS별 스크립트 | 모든 WBS에 대응 스크립트 존재 |
| 4단계 | 정합성 검증   | 검증 보고서                          | 계획서↔스크립트 일치 확인     |
| 5단계 | 스크립트 수행 | 코드 변경                            | vcxproj별 테스트 통과         |
| 6단계 | 최종 검증     | 종료 선언                            | 1~5단계 정상 수행 확인        |

## 스크립트 작성 규칙

- **위치**: `project{N}/scripts/` (또는 `x64script/` 등 작업별 디렉토리)
- **파일명**: `phase{N}-wbs{M}-{설명}.{확장자}` (예: `phase1-wbs11-dword-this.sh`)
- **내용**: 해당 WBS의 [입력] → [작업] → [출력] → [검증] 절차 구현
- **실행 환경**: 스크립트 상단에 명시 (Git Bash, PowerShell 등)
